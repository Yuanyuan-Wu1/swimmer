## 数据同步操作流程

### 1. 配置环境

```bash
# 安装必要依赖
npm install axios puppeteer cheerio node-schedule cookie

# 创建配置目录
mkdir -p backend/config
```

### 2. 获取认证信息

运行调试脚本获取West Swim Team的认证信息：
```bash
# 运行调试脚本
node backend/scripts/debugWestTeamAuth.js
```

脚本会：
1. 打开浏览器并访问West Swim Team网站
2. 自动捕获必要的认证信息
3. 将认证信息保存到 `backend/config/auth.json`

### 3. 数据同步配置

认证信息将包含：
- 必要的请求头
  ```json
  {
    "x-rio-client": "JS",
    "x-rio-entry": "coreService.loadClientModuleData",
    "x-tu-team": "pnswca",
    "x-rio-client-timezone": "America/Los_Angeles"
  }
  ```
- Cookie信息
- 其他必要的认证参数

### 4. 同步数据源

系统将从以下来源同步数据：

1. SwimCloud
   - 团队花名册: `https://www.swimcloud.com/team/8290/roster/`
   - 比赛成绩: `https://www.swimcloud.com/team/8290/times/`

2. West Swim Team
   - 比赛结果: `https://www.westswimteam.com/api/meet-results`
   - 团队数据: `https://www.westswimteam.com/api/coreService/loadClientModuleData`

### 5. 自动同步设置

系统配置为：
- 每天凌晨2点自动同步数据
- 支持手动触发同步
- 记录同步日志
- 处理数据冲突和重复

### 6. 同步数据类型

同步的数据包括：
- 选手信息
- 比赛成绩
- 比赛详情
- 团队统计数据

### 7. 错误处理

同步过程包含：
- 网络错误重试机制
- 数据验证
- 同步状态记录
- 错误通知

### 8. 维护说明

1. 认证更新
   - 认证信息可能需要定期更新
   - 运行 `debugWestTeamAuth.js` 获取新的认证信息

2. 数据验证
   - 定期检查数据一致性
   - 验证同步数据的完整性

3. 故障排除
   - 检查网络连接
   - 验证认证信息
   - 查看同步日志
   - 必要时手动触发同步

4. 配置文件
   - 认证配置: `backend/config/auth.json`
   - 环境变量: `.env`
   - 同步日志: `logs/sync.log`

### 9. 安全注意事项

- 不要提交认证信息到代码仓库
- 定期更新认证信息
- 监控异常访问
- 限制API访问频率
